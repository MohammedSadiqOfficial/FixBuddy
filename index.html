FixBuddy Application Flow Guide

Step-by-step documentation of the real-time architectural process.

1

Authentication

User and Captain sign in. The backend validates credentials and registers their Socket.io connections.

2

Service Request

User clicks "Request Service". An API POST request creates a database record, and Socket.io broadcasts the request to nearby Captains.

3

Captain Assignment

The Captain accepts the request. The backend updates the MongoDB ride status and notifies the User that a Captain is on the way.

4

Live Location Sync

The Captain app begins emitting GPS coordinates. The Node server handles the payload, bypassing heavy DB writes, and WebSockets it straight to the User's map.

5

Instant Chat

Both clients can communicate seamlessly. Messages are emitted as Socket events, instantly delivered, and asynchronously stored in MongoDB for history.

6

Admin Governance

The Admin Dashboard subscribes to global Socket rooms, passively monitoring active request, locations, and retaining the ability to reset or pause traffic.
